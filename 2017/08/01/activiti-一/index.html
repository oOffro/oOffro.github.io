activiti入门笔记(一)

发表于 2017-08-01 | 分类于 工作流 ， 入门

Activiti是工作流框架，是解决企业流程管理的解决方案。
工作流：在“多个参与者”之间按照某种“预定义的规则”传递文档、信息或任务的过程自动进行。即将现实中的工作流程，把现实生活中流程抽象为技算机流程，变成计算机能够识别流程。

工作流和工作流框架

工作流

流程：多个人在一起合作完成某件事情的步骤。
工作流：把步骤变成计算机能理解的形式就是工作流

工作流框架（工作流管理系统）

作用：

  • 定义工作流
  • 执行工作流

为什么要使用工作流框架？

传统设计由于流程变更会引起代码发生翻天覆地的改变，会影响整个项目。而使用工作流框架，可以解决这个问题。不需要改变代码，就能改变流程。

activiti组成

简介

  • 开源的工作流框架
  • 特点：开源、和jbpm有历史渊源、提供了两种方式绘制工作流程图
  • 架构：开发人员–>绘制工作流程图–>流程引擎–>部署流程–>启动流程–>办理业务

核心–工作流引擎(ProcessEngine)

服务大管家，负责生成流程运行时的各种实例及数据、监控和管理流程的运行。
用户可以通过服务完成自己想做的事情。

  1. 部署流程
  2. 启动流程
  3. 办理任务

流程定义语言–BPNM2.0

持久化

环境准备

流程设计器

  1. 安装插件activiti-eclipse-plugin5.16.rar
  2. 在bpmn文件夹中创建Activiti Diagram，画流程图

注意点： 要配置保存的时候生成一张图片
步骤：Windows->Preferences->Activiti->Save Actions->勾上 Create process…

初始化数据库

核心业务(流程实例管理)

使用Activiti提供的核心API，按照一定的通用步骤完成工作流核心业务。

  1. 启动流程
  2. 查看任务
    • 私有任务
    • 公共任务
  3. 认领任务
  4. 办理任务
  5. 查看流程状态

核心API

ProcessEngine：核心API，创建各种各样的服务

RepositoryService：仓库服务–>两个文件：bpnm文件和流程图片–>部署
RuntimeService：启动流程
TaskService：人工任务服务，查看，办理任务等操作

通用步骤

初始化步骤

  1. 导包
  2. 添加配置文件
  3. 测试

API使用步骤

  1. 获取核心对象
  2. 分析需求,通过核心对象获取对应的”服务实例”
  3. 使用获取的”服务实例”(方法)完成对应的功能

查询的步骤

  1. 创建核心对象
  2. 分析需求,获取对应的服务
  3. 通过服务,做查询
    • 创建对应查询对象
    • 添加查询条件
    • 执行查询
  4. 遍历,展示结果
